Doctors

GET /api/Doctor

Retrieves a comprehensive list of all registered doctors in the system. Typically used to populate dropdowns or doctor lists.

Request
No parameters or body required.
Response (200 OK)
Array of DoctorToReturnDto
[ { "id": 1, "fName": "Jane", "lName": "Doe", "userName": "janedoe", "gender": "Female", "country": "Egypt", "city": "Cairo", "age": 34, "graduationDegree": "/Uploads/Doctors/uuid.jpg", "medicalAssociationCard": "/Uploads/Doctors/uuid2.jpg" } ]
GET /api/Doctor/{id}

Retrieves detailed information (including document URLs) of a specific doctor by ID.

Request
Path Parameter: id (integer)
e.g., /api/Doctor/1
Response (200 OK)
DoctorToReturnDto
{ "id": 1, "fName": "Jane", "lName": "Doe", ... }
POST /api/Doctor

Creates a new doctor profile, expecting standard form fields along with document file attachments.

Request (multipart/form-data)
DoctorCreationRequest sent as Form Data:

  • NationalId: "123456789"
  • FName: "Ahmed"
  • LName: "Ali"
  • UserName: "ahmedali1"
  • Email: "ahmed@example.com"
  • Password: "Secret_123"
  • Gender: "Male"
  • Country: "Egypt"
  • City: "Alexandria"
  • Age: "42"

Files:
  • GraduationDegreeFile (.jpg/.png/.pdf)
  • MedicalAssociationCardFile (.jpg/.png/.pdf)
Response (201 Created)
Returns the created DoctorToReturnDto. Includes the new local Upload Paths generated for files.
{ "id": 2, "fName": "Ahmed", "...": "...", "graduationDegree": "/Uploads/Doctors/xxx.jpg", "medicalAssociationCard": "/Uploads/Doctors/yyy.jpg" }
PUT /api/Doctor/{id}

Allows updating primitive string/int properties for a given Doctor.

Request
Path Parameter: id
Body Type: application/json
{ "id": 1, "fName": "Jane", "lName": "Smith", "userName": "janesmith", "gender": "Female", "country": "Egypt", "city": "Cairo", "age": 35, "graduationDegree": "string_url", "medicalAssociationCard": "string_url" }
Response (204 No Content)
204 No Content Sent upon successful update.

404 Sent if the doctor ID does not exist in db.
DELETE /api/Doctor/{id}

Permanently remove a doctor from the system.

Request
Path Parameter: id (integer)
No body required.
Response (204 No Content)
204 No Content Deletion success.

Gets summarized dashbaord statistics specifically useful for a Doctor's home page.

Request

Response (200 OK)
DoctorStatisticsDto
{ "totalPatients": 12, "pendingRequests": 3 }

Request

2

Medications

GET /api/Medications/patient/{patientId}

Get isolated collection of a patient's medications independently from their full profile.

Request
Path Parameter: patientId (integer)
Response (200 OK)
Array of MedicationToReturnDto
[ { "id": 101, "name": "Donepezil", "dosage": "5mg", "frequency": "Once daily", "scheduleInstructions": "Take before bedtime", "notes": "With water", "patientId": 5, "doctorId": 2 } ]
POST /api/Medications

A Doctor prescribing a brand new medication to a specific patient.

Request
Body Type: application/json
{ "name": "Memantine", "dosage": "10mg", "frequency": "Twice a day", "scheduleInstructions": "After meals", "notes": "Start with lower half dose first week", "patientId": 5, "doctorId": 2 }
Response (201 Created)
Returns full MedicationToReturnDto.
{ "id": 102, "name": "Memantine", "dosage": "10mg", "frequency": "Twice a day", "scheduleInstructions": "After meals", "notes": "Start with lower half dose first week", "patientId": 5, "doctorId": 2 }
DELETE /api/Medications/{id}

Revoke a medication prescription from a patient's active lineup.

Request
Path Parameter: id (integer)
No body required.
Response (204 No Content)
204 No Content Success removal.

Mind Games

GET /api/MindGames/catalog

Retrieves globally available mind games stored securely in the app used for assigning to users.

Request
No parameters needed.
Response (200 OK)
Array of MindGameDto
[ { "id": 1, "name": "Word Puzzle", "description": "Crossword for logic.", "category": "Vocabulary", "thumbnailUrl": "/assets/games/word.png" } ]
GET /api/MindGames/patient/{patientId}

Retrieves the specific list of active mind games currently bound to the patient.

Request
Path Parameter: patientId
Response (200 OK)
Array of PatientMindGameDto
[ { "patientId": 5, "mindGameId": 1, "mindGame": { "id": 1, "name": "Word Puzzle", ... }, "assignedOn": "2024-03-12T10:00:00" } ]
POST /api/MindGames/patient/{patientId}/assign/{gameId}

Used to append a particular mind game to a patient's routine list. Only requires parameters on the URL path.

Request
Path Parameters:
patientId: 5
gameId: 1

Body: None required.
Response (200 OK)
JSON Message payload.
{ "message": "Game assigned successfully" }
DELETE /api/MindGames/patient/{patientId}/remove/{gameId}

Removes an assigned mind game from a patient's active games list.

Request
Path Parameters:
patientId: 5
gameId: 1

Body: None required.
Response (204 No Content)
204 No Content Success removal.

Treatment Requests

POST /api/TreatmentRequests

Form submission containing detailed behavioral symptoms linked to a patient sent for a doctor's review.

Request
Body: TreatmentRequestToCreateDto
{ "memoryLossFrequency": "Daily", "confusionFrequency": "Weekly", "languageProblems": "Often struggles to name objects", "moodChanges": "Easily agitated", "repetitiveBehavior": "Repeats same questions", "patientId": 5, "doctorId": 2, "relativeId": 12 }
Response (200 OK)
TreatmentRequestToReturnDto
{ "id": 50, "memoryLossFrequency": "Daily", "confusionFrequency": "Weekly", "languageProblems": "Often struggles to name objects", "moodChanges": "Easily agitated", "repetitiveBehavior": "Repeats same questions", "status": "Pending", "patientId": 5, "patientName": "Sarah Connor", "patientAge": 68, "doctorId": 2, "relativeId": 12, "relativeName": "John Connor", "relativeRelationship": "Son" }
GET /api/TreatmentRequests/doctor/{doctorId}/pending

Return an array of all treatment requests pending action by the specified doctor.

Request
Path Parameter: doctorId
Response (200 OK)
Array of pending TreatmentRequestToReturnDto.
[ { "id": 50, "status": "Pending", "patientName": "Sarah Connor", ... } ]

Update the state of a specific TreatmentRequest to 'Accepted'. This binds the patient to the doctor formally in the system.

Request
Path Parameter: id (integer)

No body required.
Response (204 No Content)
204 No Content Success.

Request
Path Parameter: id (integer)

Response (204 No Content)
